# Asterisk Config File Generation

## Scope

Writing non-dialplan Asterisk configuration files during `retrieve_conf`. BMO: `genConfig()`/`writeConfig()`. Legacy: `{mod}_conf` classes. Written via `WriteConfig` class to `/etc/asterisk/`.

---

## Trigger Flow

```
retrieve_conf
  → FileHooks::processFileHooks($active_modules)
    → processOldHooks()   // legacy {mod}_conf classes
    → processNewHooks()   // BMO genConfig/writeConfig
```

Modules discovered via `Hooks::updateBMOHooks()` — any BMO class with `writeConfig()` method added to `ConfigFiles` cache.

---

## BMO Pattern

### `genConfig()` — Generate Config Data

```php
public function genConfig()
{
	return [
		'mymodule.conf' => [
			'[general]',
			'enabled=yes',
			'timeout=30',
		],
		'mymodule_custom.conf' => "; custom settings\n",
	];
}
```

Return format: `[filename => string|array of lines]`

### `writeConfig($config)` — Write Files

```php
public function writeConfig($config)
{
	$this->FreePBX->WriteConfig($config);
}
```

`$config` is the array returned by `genConfig()` (possibly modified by other modules' hooks).

---

## `WriteConfig` Class API

```php
// Single file
FreePBX::WriteConfig()->writeConfig('mymodule.conf', $contents, $generateHeader = true);

// Multiple files
FreePBX::WriteConfig()->writeConfigs([
	'file1.conf' => ['line1', 'line2'],
	'file2.conf' => "string contents",
]);

// Constructor batch write
new \FreePBX\WriteConfig($freepbx, ['file.conf' => $data]);
```

### Auto-Generated Header

```ini
;--------------------------------------------------------------------------------;
;          Do NOT edit this file as it is auto-generated by FreePBX.             ;
;--------------------------------------------------------------------------------;
```

Set `$generateHeader = false` to omit.

### File Path Validation

- Files written to `/etc/asterisk/`
- Filenames must not contain `/` (except `digium_phones/` prefix allowed)
- `..` in filename throws Exception

---

## `FileHooks::processNewHooks()` Sequence

```php
foreach ($hooks['ConfigFiles'] as $hook) {
	$tmpconf = $this->FreePBX->$hook->genConfig();
	// Hook interception point for $tmpconf (not yet implemented)
	$this->FreePBX->$hook->writeConfig($tmpconf);
}
```

Missing `genConfig()` or `writeConfig()` throws Exception.

## Legacy `{mod}_conf` Pattern

```php
// Auto-loaded class: mymodule_conf
class mymodule_conf {
	public $use_warning_banner = true;

	function get_filename() {
		return 'mymodule.conf';
		// or array('file1.conf', 'file2.conf') for multiple
	}

	function generateConf($filename = null) {
		return ["[general]\n", "enabled=yes\n"];
	}
}
```

### Legacy Dispatch

```php
$classname = $mod . "_conf";
if (class_exists($classname) && method_exists($classname, 'get_filename')) {
	$module = method_exists($classname, 'create') ? $classname::create() : new $classname;
	$fn = $module->get_filename();
	// Single or array of filenames
	WriteConfig->writeConfig($fn, $module->generateConf($fn), $generateHeader);
}
```

`use_warning_banner = false` suppresses FreePBX header.

---

## Config Array Formats

| Format | Example | Written As |
|--------|---------|------------|
| String | `"; comment\nkey=val\n"` | Literal file content |
| Array of lines | `['[section]', 'key=val']` | Lines joined with newlines |
| Mixed per file | `['a.conf' => 'str', 'b.conf' => ['lines']]` | Per-file format |

## Realtime Config Example

```php
// Realtime.class.php pattern
$conf = [/* generated */];
$this->FreePBX->WriteConfig($conf);
```

Shortcut when already holding config array.

## `ConfigFile` BMO Class

Higher-level wrapper for structured config building:

```php
// ConfigFile.class.php
$this->FreePBX->WriteConfig([$this->file => $this->config->ProcessedConfig]);
```

Used by modules needing validated config object pipeline.

---

## Custom File Writing

```php
FreePBX::WriteConfig()->writeCustomFile([
	'/custom/path/file.conf' => $contents,
], $generateHeader = false);
```

Bypasses `/etc/asterisk/` restriction for explicitly pathed files.

---

## Hook Integration (Future)

`FileHooks` contains placeholder for handing `$tmpconf` to other modules before `writeConfig()` — not yet implemented. Plan for config modification hooks if adding cross-module config injection.

---

## Include in asterisk.conf

Generated files must be `#include`d in `asterisk.conf` or relevant driver config. FreePBX core modules manage base includes; new modules may need:

1. Write file via `writeConfig()`
2. Ensure `includes` module or custom logic adds `#include mymodule.conf`

---

## Constraints

- Files overwritten on every `retrieve_conf` — local edits lost
- Filename must not contain path traversal
- `genConfig()` must return array (even for single file)
- `writeConfig()` must call `FreePBX->WriteConfig()` — do not write files directly with `file_put_contents`
- Module must have `writeConfig()` method to appear in `ConfigFiles` cache
- Legacy and BMO patterns can coexist during migration
- Test with `fwconsole reload` and verify file contents in `/etc/asterisk/`
